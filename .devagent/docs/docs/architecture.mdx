# Architecture

scryrs is a Rust workspace split by product capability: the CLI composes feature-gated crates, the deterministic core owns trace and hotspot logic, guardrail crates bound risky behavior, and adapters publish knowledge to documentation systems without making any single docs framework mandatory.

The key architectural separation: **suite crates own product capabilities**, **guardrail crates bound unsafe or privacy-sensitive behavior**, and **adapters are publishing surfaces**. `crates/scryrs-cli` composes those pieces through Cargo features; `crates/scryrs-types` keeps cross-crate contracts small and explicit.

The LLM layer is not in the policy path. `crates/scryrs-llm` is an optional transport boundary for explanation, suggestion ranking, or patch drafting. Deterministic crates produce evidence first; model-backed features can interpret that evidence later.

## Crate Responsibilities

| Crate | Role | Why it exists |
| --- | --- | --- |
| `crates/scryrs-cli` | Main `scryrs` binary | Keeps user, CI, and agent command behavior in one feature-gated shell. |
| `crates/scryrs-types` | Shared contracts | Prevents circular dependencies and keeps cross-crate data shapes explicit. |
| `crates/scryrs-core` | Trace and hotspot foundation | Preserves standalone hotspot detection without docs-site assumptions. |
| `crates/scryrs-graph` | Knowledge graph foundation | Owns routing graph concepts separate from trace collection. |
| `crates/scryrs-curator` | Proposal foundation | Turns hotspots into reviewable docs, skill, decision, or memory proposals. |
| `crates/scryrs-runtime` | Agent-side routing foundation | Helps future agents choose relevant context faster. |
| `crates/scryrs-adapter-markdown` | Markdown publishing adapter | Supports generic docs repos and simple file-based output. |
| `crates/scryrs-adapter-rspress` | Rspress publishing adapter | Makes Rspress first-class without making it foundational. |
| `crates/scryrs-policy` | Deterministic guardrail decisions | Keeps policy decisions in code, not prompts. |
| `crates/scryrs-sandbox` | Tool and filesystem capability policy | Establishes read/write/exec boundaries before tool execution exists. |
| `crates/scryrs-telemetry` | Privacy and telemetry defaults | Makes observability opt-in and redaction-first. |
| `crates/scryrs-llm` | Optional model boundary | Keeps provider/model transport replaceable and outside deterministic policy. |
| `xtask` | Repository automation | Provides a stable place for bootstrap, CI, release, and generated-artifact tasks. |

## Cargo Feature Model

`crates/scryrs-cli/Cargo.toml` is the composition point.

```text
default
= core + graph + curator + markdown + runtime + guardrails

guardrails
= policy + sandbox + telemetry

full
= default + llm + rspress
```

This lets users keep the CLI lean for standalone hotspot analysis while enabling richer suite behavior when needed:

```bash
cargo run -p scryrs-cli -- hotspots .
cargo run -p scryrs-cli --no-default-features --features core -- hotspots .
cargo run -p scryrs-cli --features full -- hotspots .
```

Policy comes before model use. Sandbox and telemetry crates exist even before full implementations so future features have obvious homes and do not smuggle shell access, write access, or remote logging into product crates.

## Publishing Surfaces

scryrs does not replace a documentation system. It tells the docs system what it should know.

Adapters depend on shared contracts and suite outputs. Core detection should never depend on Rspress, Markdown layout, or a published docs site.

## Source-of-Truth Rules

- `Cargo.toml` defines workspace membership, shared dependency aliases, and workspace lint policy.
- `crates/scryrs-cli/Cargo.toml` defines the product feature matrix.
- `crates/scryrs-types/src/lib.rs` owns shared contracts used across suite crates.
- `.devagent/architecture/` owns LikeC4 source for embedded architecture diagrams on this page.
- `scripts/check`, `scripts/test`, and `scripts/security` are local guardrail entry points.
- `.github/workflows/ci.yml` mirrors fast-lane checks for pull requests and `main`.
- `deny.toml`, `clippy.toml`, `rustfmt.toml`, and `rust-toolchain.toml` capture tool policy.

## Current Limitations

- Behavior is scaffold-level: commands print placeholders and crates expose minimal public contracts.
- LikeC4 views on this page are high-level architecture projections, not exhaustive Cargo dependency graphs.
- Rust toolchain is not installed in the current local environment, so cargo-backed verification is blocked until `rust-toolchain.toml` can be installed.
